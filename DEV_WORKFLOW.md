# Development Workflow (STRICT)

You MUST follow this workflow:

---

## Step 1: Requirement Analysis
- Restate the problem
- Identify missing information
- Clarify assumptions

---

## Step 2: Specification
- Generate a structured spec using SPEC_TEMPLATE.md

---

## Step 3: Task Decomposition
- Break into executable tasks using TASK_TEMPLATE.md

---

## Step 4: Implementation
- Implement step by step
- Explain key decisions

---

## Step 5: Review
- Use REVIEW_TEMPLATE.md
- Identify risks, bugs, improvements

---

## Rules

- DO NOT skip steps
- DO NOT jump directly to coding
- ALWAYS think before acting
